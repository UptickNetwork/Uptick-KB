## 引言
Uptick Network 正在為非同質化代幣（NFT）建立企業級的基礎設施與生態系統。
該平台的設計聚焦於多鏈與跨鏈互通性，包含三大核心組件：NFT 基礎設施、NFT 交易市場以及 NFT 生態應用。

### Uptick 鏈
Uptick 鏈是基於 Cosmos-SDK 構建的區塊鏈網絡，是 Uptick 基礎設施的核心基石。
它是專門為支援 NFT 的使用而設計的。

### IBC 與 EVM 支持
平台整合了對以太坊虛擬機器（EVM）以及部分跨鏈通訊（IBC）網路的支援。
致力於為各類實用且具開創性的 NFT 相關場景提供必要的互通工具與服務。

### 基礎設施
Uptick 基礎設施在 Uptick 鏈之上提供了一套先進的模組化商業協議。
這些協議由擁有豐富區塊鏈產業經驗的團隊開發，旨在支援各類與商業相關的 NFT 功能與特性，其中部分包括：
- 可互通的 ERC 與 CW 資產標準
- 透過 IBC 與 EVM 網路實現的跨鏈 NFT
- 面向 NFT 經濟的去中心化金融（DeFi）協議
- 跨鏈 NFT 元資料標準
- 去中心化資料即服務

Uptick Network 是基於 Cosmos-SDK 構建，並由 Tendermint 共識引擎提供安全保障。
它被歸類為應用型區塊鏈，針對更廣泛的區塊鏈生態系統中的特定應用與使用場景進行了最佳化。

此外，Uptick Network 在更廣闊的 Cosmos 生態系統中作為一個獨立的區塊鏈專區運作。
它遵循 Cosmos 通用的通證效用與激勵模型，同時也針對 NFT 相關基礎設施、生態發展以及擴展的 DAO 治理增加了自身的創新支持。

## 通證效用
Uptick Network 的願景是為商業世界中高品質 NFT 的創建與使用提供基礎架構。這包括必要的底層區塊鏈技術，以及各類其他工具，以支撐多種不同的商業環境。

關於 Uptick Network 具體的通證經濟模型，請參考通證經濟學章節。

由 Uptick 鏈管理的通證並不局限於 Cosmos-SDK 支援的原生通證。它們還包括基於 ERC-20 的通證以及 NFT。 Uptick Network 開發了一套費用模型，不僅與 Cosmos-SDK 相容，同時也適配 EVM，為各類通證提供彈性。

UPTICK 是 Uptick 鏈上使用的原生平台通證，其效用主要分為四大類：
- 手續費
- 質押
- 通用治理
- 擴展型基於規則的治理

### 手續費
Uptick Network 使用其原生 UPTICK 通證設定交易手續費，該費用作為驗證者將交易打包進下一個區塊的激勵。每筆交易的平均手續費約為 0.001 UPTICK。

在 Uptick 鏈正式上線的節點中，基於 EVM 的費用（EVM 費用）與基於 Cosmos-SDK 的費用（Cosmos 費用）保持基本一致。

Uptick 透過 feemarket 模組計算 EVM 費用與 Cosmos 費用。

#### 計算公式

| fee = (baseFee + priorityTip) \* gasLimit |
| --- |

其中 baseFee 是每個 gas 單位的固定區塊網路費用，priorityTip 是可選設定的每 gas 單位額外費用。請注意，基本費用與優先小費均為 gas 價格。若要提交符合 EIP-1559 的交易，簽署者需要指定 gasFeeCap，即其願意支付的每 gas 單位總費用上限。可選地，可以指定 priorityTip，該費用同時包含優先費用與每個 gas 單位的區塊網路費用（即基礎費用）。

| Cosmos-SDK 對 gas 的術語定義與以太坊不同。以太坊中稱為 gasLimit 的概念，在 Cosmos 中稱為 gasWanted。在 Uptick 上你可能會同時遇到這兩個術語，因為它在 SDK 之上建立了以太坊相容層，例如在使用不同錢包時，Keplr 對應 Cosmos，MetaMask 對應以太坊。 |
| --- |

#### 參數設定
| 鍵名 | 類型 | 初始值 | 說明 |
| --- | --- | --- | --- |
| NoBaseFee | bool | FALSE | 控制基礎費用的調整 |
| BaseFeeChangeDenominator | uint32 | 7 | 限制區塊之間基礎費用的變動幅度 |
| ElasticityMultiplier | uint32 | 2 | 限制基礎費用依據上一區塊總 gas 使用量進行增減的門檻 |
| BaseFee | uint32 | 1000000000 | EIP-1559 區塊的基礎費用 |
| EnableHeight | uint32 | 0 | 啟用費用調整的區塊高度 |
| MinGasPrice | sdk.Dec | 0 | 交易打包進區塊所需支付的全域最低 gas 價格 |

#### 手續費分配
上述交易手續費除分配給權益證明（PoS）節點與質押帳戶外，也會以社區稅的形式收取並進入社區池。
社區池中的部分資金將作為通證經濟模型的一部分，用於參與社區激勵，或直接銷毀，作為控制通膨的手段之一。

更多細節請參考通證經濟學章節。

#### 質押獎勵分配

非營利組織的質押，例如基金會與生態發展相關質押。

**普通用戶質押**

用戶可依自身偏好，在質押期限到期（21 天）後選擇繼續質押或解鎖。

**非營利組織質押（如基金會與生態發展）**

基於治理投票結果，這些資金將進入擴展型基於規則的治理的激勵池，用於激勵 Uptick 去中心化應用（dApp）生態與多鏈經濟。

**社區池分配**

一部分質押獎勵（初始設定為 2%）將進入社區池。該池將用於激勵對 Uptick 生態做出貢獻的項目，具體由社區投票決定。

此外，部分通證可能會被銷毀，以控制通膨與通證流通量。

### 通用治理

治理的角色是管理 Uptick 生態，確保其永續與健康發展。
透過治理機制，社群成員可以參與協議的製定與修改，以及網路上提案的投票。

治理的功能包括但不限於：
- 維護 Uptick 協定的完整性與穩定性
- 定義與修改協議參數與規則，例如通膨率、交易手續費、治理參數
- 投票分配與使用社區內資金，包括指定用於開發項目、慈善事業及其他社區舉措的資金
- 投票支持或反對升級或變更提案，例如軟體版本更新或新功能引入
- 鼓勵社區成員積極參與社區治理，增強社區共識與參與度，推動 Uptick 生態發展

#### 治理功能
Uptick 的治理可透過社群投票執行，包括但不限於：
- 透過調整參數設定表格中的內容，控制原生 Uptick 通證的通膨率
- 透過投票銷毀一部分通證
- 透過投票來確定獎勵金額並制定擴展型基於規則的治理的獎勵規則

### 擴展型基於規則的治理

由於 Uptick Network 內建 EVM 模組，UPTICK 通證以兩種格式存在於網路中：
- Cosmos 原生通證
- ERC-20 相容通證

透過 EVM（後續也將支援 CosmWasm 及更多虛擬機），Uptick 鏈支援基於智慧合約的通證管理方式。
因此，除 Cosmos-SDK 模組定義的預設方式外，還將有多種創新方式建構類 DAO 治理規則。

Uptick 鏈將逐步實現以下鏈上加密資產的社群化治理，而不僅限於平台通證：
- 原生平台通證：UPTICK
- IBC 轉帳通證
- 生態應用發行的 ERC-20 相容通證
- 具備公允市價的 NFT 資產

請注意，UPTICK 通證的效用將進一步完善，以確保符合法規要求。

## 通證供應

UPTICK 通證初始發行量為 10 億枚（1,000,000,000）。

由於權益證明（PoS）網路的特性，Uptick 鏈的年通膨率將根據質押率與銷毀率的變化，在 4% 至 10% 之間動態波動。

UPTICK 通證設計為價值儲存型通證，透過在 SDK 鏈與智慧合約層級實施的鎖倉機制，有效控制市場流通。

## 初始通證分配

初始分配結構旨在激勵長期持有者與 Uptick Network 生態。

UPTICK 通證的分配計畫如下：

**私募與顧問：15%**

從 Uptick Network 上線開始，設定為期一年的歸屬期，期間 UPTICK 通證以每日 1/365 的比例線性歸屬。
全部通證存放於歸屬帳戶中，持有者可使用未歸屬通證在 Uptick Network 上進行質押。

該措施旨在提升 Uptick Network 的安全性，同時允許 UPTICK 持有者獲得區塊獎勵。

**核心團隊：15%**

從 Uptick Network 上線開始，設定為期四年的歸屬期，團隊每六個月解鎖其 UPTICK 通證的 1/8。
全部通證由多簽歸屬帳戶保管，持有者可使用未歸屬通證在 Uptick Network 上進行質押。

該措施旨在提升 Uptick Network 的安全性，同時允許團隊獲得區塊獎勵。

**基金會：20%**

預留用於支持基金會的運作。

**生態發展：47.5%**

- 與策略跨鏈夥伴進行價值交換
- 生態應用建構者與終端用戶的績效型激勵（應用開發與營運挖礦）
- 優秀夥伴獎勵
- NFT 質押與借貸激勵
- 跨鏈去中心化金融挖礦
- 針對 Uptick DAO 參與者與活動的激勵與獎勵
- 其他用途

**空投：2.5%**

本次空投的目標是支持 Uptick Network 及其核心生態合作夥伴與用戶的長期成功。
其涵蓋範圍包括但不限於以下類別：
- 媒體與市場推廣貢獻者
- 測試網驗證者與治理貢獻者（含測試網 1.0 與測試網 2.0）
- 測試網市場貢獻者（含測試網 1.0 與測試網 2.0）
- Uptick 的 IRISnet 市場貢獻者
- Uptick 的 Loopring 市場貢獻者
- 符合特定條件的 Cosmos Hub ATOM 通證持有者
- 符合特定條件的 IRISnet Hub IRIS 通證持有者
- IRISnet 治理中支持 Uptick 相關提案的支持者
- 其他支援 Uptick 計畫的平台、應用與社區

空投將分階段發放，而非在創世啟動日一次全部發放。
此外，部分類別的空投將遵循歸屬計畫。

## 通證經濟學

區塊鏈專案難以落地的痛點之一在於，許多公鏈的通證經濟僅聚焦於鏈本身，只考慮零層或一層的經濟模型，而忽略了通證經濟對生態應用與跨鏈生態的支撐。

Uptick Network 基於上述問題設計了多層級通證經濟體系，包括：
- Uptick 網路基礎層
負責 UPTICK 通證的發行與流通。
此層級的通證經濟設計旨在保障 UPTICK 通證的穩定與安全。

- Uptick 去中心化應用生態層
包括所有建構在 Uptick Network 之上的去中心化應用程式。
此層級的通證經濟設計旨在激勵開發者建構並貢獻於 Uptick 去中心化應用生態。

- 多鏈經濟層
不同區塊鏈網路之間的跨鏈合作與互動。
此層級的通證經濟設計旨在激勵並促進跨鏈交易與合作。

- 社區層
Uptick Network 社群成員與利害關係人。
此層級的通證經濟設計旨在激勵社區參與與貢獻，確保 Uptick Network 的永續性與成長。

透過設計多層級通證經濟，Uptick Network 旨在打造全面且永續的生態系統，惠及所有利害關係人，包括用戶、開發者、投資者及更廣泛的區塊鏈社群。

多層級通證經濟具備多項優勢：
**完整性**

通證經濟的多層架構兼顧各類商業場景，分配鏈、跨鏈、生態等各方利益。

**可擴展性**

Uptick DAO 治理與多鏈經濟模型基於圖靈完備的 EVM 合約，理論上可實現任意通證經濟邏輯。

**安全性**

通用治理基於 Cosmos-SDK，該框架已通過 Cosmos 生態驗證，保障通證經濟模型的安全。
此外，Uptick DAO 治理與多鏈經濟模式的安全性由底層 EVM 安全機制所保障。
與去中心化金融應用不同，Uptick Network 經濟模型合約的合約帳戶為 Cosmos 模組帳戶，可避免因私鑰外洩而導致的安全風險。

**去中心化**

經濟模型規則與修改透過 Cosmos-SDK 治理模組執行，需多數質押者確認。
此去中心化方式確保決策不會被單一實體控制。

### 通用治理
儘管 Cosmos 生態中沒有對零層與一層的明確定義，但 Cosmos-SDK 可被視為等效於零層。
Cosmos-SDK 是一套支援在 Cosmos 生態中建構中心樞紐與專區的框架。

它已被證明在實現去中心化治理方面安全且有效。
因此，Uptick Network 選擇使用 Cosmos-SDK 來保障其通證經濟的安全性、有效性與去中心化治理。

**動態質押模型**

Uptick Network 採用動態質押模型，根據目前質押的 UPTICK 通證比例調整年通膨率。
質押通證越多，通膨率越低，有助於維持通證價值。

此外，部分交易手續費將被銷毀，減少 UPTICK 總供應量，提升剩餘通證的價值。

UPTICK 通證持有者可參與網路治理決策，例如對網路升級與參數變更提案進行投票。
這讓使用者在 Uptick Network 的發展與方向上擁有話語權。

### Uptick DAO 治理（擴展型基於規則的治理）
Uptick DAO 是一套在 Uptick 生態中使用 EVM 合約建構的基於規則的智慧合約集合。儘管合約是基於 EVM，但其規則（合約產生）與修改（合約升級）需要透過底層 Uptick 驗證者投票驗證。

該合約同時保障 EVM 合約強大的去中心化金融功能與 Cosmos 體系的去中心化管理模式。

- 基金會與生態發展資金的質押獎勵將捐贈至生態應用獎勵池（EARP）
- 任何驗證者也可將部分佣金/獎勵捐贈至生態應用獎勵池，以支持生態應用
- 生態應用獎勵池資金的分配是基於績效衡量指標，即一套生態應用獎勵池規則

範例：
- 應用可參與生態應用獎勵池的質押獲利模式吸引使用者。
透過獲取利息獎勵與應用程式內會員特權，鼓勵用戶在該應用通道質押通證。
- 應用的總鎖倉價值（TVL）越高，為 Uptick 生態帶來的價值越大。
- 應用可參與生態應用獎勵池的交易挖礦模式吸引使用者。
此應用通道產生的交易將由基礎架構上的資料分析器記錄與分析。
- 應用可參與生態應用獎勵池的 NFT 質押獲利模式吸引用戶。
NFT 的估值應獲得市場的公平認可。

### 通膨控制
在權益證明（PoS）區塊鏈中，通膨控制透過多種機制實現，例如調整區塊獎勵與通證供應量。
對於 Uptick Network，有多種方式控制通膨：

**區塊獎勵調整**

區塊獎勵是驗證者每驗證一個區塊所獲得的通證數量。
透過調整區塊獎勵，可以控制通膨率。
在 Uptick Network 中，區塊獎勵由通證總供應量與目標年通膨率決定。

**動態質押獎勵**

Uptick Network 也支持動態質押獎勵，根據質押通證數量調整獎勵。
這意味著質押通證越多，獎勵越低，反之亦然。
此機制旨在鼓勵更多通證持有者質押通證，以協助網路安全。

**社群驅動治理**

Uptick Network 基於社群驅動治理模型，通證持有者可提案並投票決定網路變更，包括通膨率調整。
透過社區驅動治理，可調整通膨率以反映社區的需求與意願。

透過謹慎平衡這些機制，網路可長期維持穩定且可持續的通膨率。
